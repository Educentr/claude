# Add Telegram Bot to Existing Project

Interactive command to add a Telegram bot worker to an existing go-project-starter project.

## Mandatory Steps

The task is NOT complete until ALL of these pass:
1. Read existing project.yaml
2. Interview the user if needed
3. Modify project.yaml
4. Run `make regenerate` successfully
5. Run `make build` successfully (no compilation errors)

## Step 1: Read Existing Config

Find and read the project config:
```bash
# Usually in .project-config/project.yaml
cat .project-config/project.yaml
```

Identify:
- The `module_path` from git section (needed for driver import)
- Existing applications, transports, workers, drivers
- Whether sys transport already exists

## Step 2: Interview

Ask the user (skip if obvious from context):

1. **Separate container?** — run bot as separate app (recommended) or in existing app?
2. **Bot name** — worker name, e.g., `telegrambot` (default)
3. **Need payments?** — include UnimplementedPayment stub?

## Step 3: Modify Config

Add these sections to `project.yaml`:

### Add worker (if not exists):
```yaml
worker:
  - name: telegrambot
    generator_type: template
    generator_template: telegram
```

### Add driver (if not exists):
```yaml
driver:
  - name: telegram
    import: {module_path}/pkg/drivers/telegram
    package: telegram
    obj_name: Telegram
    service_injection: |
      telegram.BaseAuth
      telegram.UnimplementedPayment
```

**CRITICAL:** The `import` path MUST use the project's own `module_path` from git section, NOT a hardcoded path.

### Add new application (if separate container):
```yaml
applications:
  # ... existing apps unchanged ...
  - name: telegram-bot
    transport:
      - name: sys      # sys MUST already exist in rest section
    worker: [telegrambot]
    driver:
      - name: telegram
        params: [WithUpdatePoller()]
```

### Or add to existing application:
```yaml
applications:
  - name: server
    transport:
      - name: api
      - name: sys
    worker: [telegrambot]          # Add worker
    driver:
      - name: telegram             # Add driver
        params: [WithUpdatePoller()]
```

### Validation checklist before saving:
- [ ] Worker name is unique (no duplicates)
- [ ] Driver name is unique
- [ ] Driver import uses correct module_path
- [ ] Bot application has at least one transport (sys)
- [ ] sys transport is defined in rest section
- [ ] All entities referenced in applications exist

## Step 4: Regenerate

```bash
make regenerate
```

If fails, read error, fix config, retry.

## Step 5: Build

```bash
make build
```

If fails, fix and retry. Common issues:
- Missing sys transport definition
- Wrong driver import path
- Name conflicts

Only report success when build passes.

## Step 6: Summary

Tell the user:
- What was added to project.yaml
- Bot handler location: `internal/app/worker/telegrambot/handler/handler.go`
- Router: `internal/app/worker/telegrambot/router.go`
- Driver: `pkg/drivers/telegram/`
- OnlineConf token path: `/{service_name}/worker/telegram/telegrambot/token`
- Worker is DISABLED by default — must set enabled=true in OnlineConf
- Run: `make run-telegram-bot` (if separate app)
