# Add REST API to Existing Project

Interactive command to add a REST API transport (server or client) to an existing go-project-starter project.

## Mandatory Steps

The task is NOT complete until ALL of these pass:
1. Read existing project.yaml
2. Interview the user
3. Modify project.yaml (and create OpenAPI spec if needed)
4. Run `make regenerate` successfully
5. Run `make build` successfully (no compilation errors)

## Step 1: Read Existing Config

```bash
cat .project-config/project.yaml
```

Identify:
- Existing REST transports (names, ports, versions)
- Existing applications and their transport lists
- The `module_path` from git section

## Step 2: Interview

Ask the user:

1. **Server or client?**
   - `ogen` — REST API server from OpenAPI spec (handle incoming requests)
   - `ogen_client` — REST client for calling external API
2. **Transport name** — unique name (e.g., `admin_api`, `payment_client`)
3. **OpenAPI spec** — does the user have a swagger file? Path to it?
4. **Port** — HTTP port (for servers). Must not conflict with existing ports.
5. **API version** — default `v1`
6. **Auth** — for servers: `auth_handler: "on"/"off"`? For clients: `auth_params` needed?
7. **Which application?** — add to existing app or create new one?
8. **Instantiation** (clients only) — `static` (default) or `dynamic`?

## Step 3: Modify Config

### Adding ogen server:

Add to `rest` section:
```yaml
rest:
  # ... existing transports ...
  - name: {transport_name}
    path: [./api/{transport_name}.swagger.yml]
    generator_type: ogen
    port: {port}
    version: v1
    generator_params:
      auth_handler: "{on_or_off}"
```

If user doesn't have an OpenAPI spec, create a minimal one at the path specified. The spec MUST include ErrorDefault schema:

```yaml
openapi: "3.0.0"
info:
  title: "{transport_name} API"
  version: "1.0.0"
paths:
  /health:
    get:
      operationId: healthCheck
      responses:
        '200':
          description: OK
          content:
            application/json:
              schema:
                type: object
                properties:
                  status:
                    type: string
        default:
          description: Error
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorDefault'
components:
  schemas:
    ErrorDefault:
      required: [code, error]
      properties:
        code:
          type: integer
          format: int32
        error:
          type: string
      type: object
```

### Adding ogen_client:

```yaml
rest:
  # ... existing transports ...
  - name: {client_name}
    path: [./api/{client_name}.swagger.yml]
    generator_type: ogen_client
    # For static (default):
    port: 443
    version: v1
    # For dynamic:
    instantiation: dynamic
    # For auth:
    auth_params:
      transport: header
      type: apikey        # or "bearer"
```

### Add to application:

```yaml
applications:
  - name: server
    transport:
      # ... existing transports ...
      - name: {transport_name}
      # For client with per-app override:
      - name: {client_name}
        config:
          instantiation: dynamic    # optional override
          optional: true            # optional: won't fail if unavailable
```

### Validation checklist:
- [ ] Transport name is unique (no duplicates in rest section)
- [ ] Port doesn't conflict with existing transports
- [ ] OpenAPI spec file exists at specified path
- [ ] ErrorDefault schema present in spec (for ogen servers)
- [ ] Transport added to at least one application
- [ ] `instantiation` only used with `ogen_client`

## Step 4: Regenerate

```bash
make regenerate
```

If fails, read error, fix config, retry.

## Step 5: Build

```bash
make generate    # Runs ogen code generation
make build
```

Common build errors:
- `undefined: oas.ErrorDefault` — add ErrorDefault schema to OpenAPI spec
- `cannot use &Handler{} as type oas.Handler` — generated handler interface mismatch, run `make generate` again
- Port conflicts — change port in config

Fix any errors and retry until build passes.

## Step 6: Summary

### For ogen server:
- Handler location: `internal/app/transport/rest/{name}/v1/handler/`
- Router: `internal/app/transport/rest/{name}/v1/psg_router_gen.go`
- Implement handlers in `handler/*.go` (extend generated `psg_handler_gen.go`)
- Your Handler struct embeds `oas.UnimplementedHandler` — override methods you need
- Disclaimer marker separates generated from manual code

### For ogen_client:
- Generated client: `pkg/rest/{name}/v1/`
- Client middleware: `internal/app/transport/rest/{name}/v1/mw/`
- OnlineConf paths:
  - `/{service}/transport/rest/{name}_v1/host` — API host
  - `/{service}/transport/rest/{name}_v1/auth_params/apikey` — API key (if apikey auth)
  - `/{service}/transport/rest/{name}_v1/auth_params/token` — bearer token (if bearer auth)
