# Add Kafka to Existing Project

Interactive command to add Kafka producer or consumer to an existing go-project-starter project.

## Mandatory Steps

The task is NOT complete until ALL of these pass:
1. Read existing project.yaml
2. Interview the user
3. Modify project.yaml (and optionally create JSON Schema files)
4. Run `make regenerate` successfully
5. Run `make build` successfully (no compilation errors)

## Step 1: Read Existing Config

```bash
cat .project-config/project.yaml
```

Identify existing applications, kafka configs, jsonschema sections.

## Step 2: Interview

Ask the user:

1. **Producer or consumer?** — `type: producer` or `type: consumer`
2. **Name** — unique name, e.g., `events_producer`, `order_consumer`
3. **Client name** — Kafka client identifier for OnlineConf paths (e.g., `main_kafka`)
4. **Events** — list of event names (e.g., `user_created`, `order_updated`)
5. **Typed messages?** — use JSON Schema for typed Go structs? If yes:
   - Schema file paths or create new ones
   - Schema set name and package name
6. **Consumer group** — required for consumers (e.g., `my_consumer_group`)
7. **Which application?** — add to existing app or new one?

## Step 3: Modify Config

### Add jsonschema (if typed messages):
```yaml
jsonschema:
  - name: {schema_set_name}
    package: {package_name}    # optional, defaults to name
    schemas:
      - id: {event_schema_id}
        path: ./schemas/{event_name}.schema.json
```

Create the JSON Schema files:
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "id": {"type": "integer"},
    "name": {"type": "string"},
    "timestamp": {"type": "string", "format": "date-time"}
  },
  "required": ["id"]
}
```

### Add kafka section:

**Producer:**
```yaml
kafka:
  - name: {producer_name}
    type: producer
    driver: segmentio
    client: {client_name}
    events:
      - name: {event_name}
        schema: {schema_set}.{schema_id}    # Only if typed messages
```

**Consumer:**
```yaml
kafka:
  - name: {consumer_name}
    type: consumer
    driver: segmentio
    client: {client_name}
    group: {consumer_group}           # Required for consumers!
    events:
      - name: {event_name}
        schema: {schema_set}.{schema_id}
```

### Add to application:
```yaml
applications:
  - name: server
    transport: [...]
    kafka:
      - {kafka_name}                  # Simple string format (required)
    # OR with optional flag:
    kafka:
      - name: {kafka_name}
        optional: true                # Won't fail if Kafka unavailable
```

### Validation checklist:
- [ ] Kafka name is unique
- [ ] Consumer has `group` field
- [ ] `client` field is set
- [ ] Events list is not empty
- [ ] Schema references (`schema_set.schema_id`) match existing jsonschema definitions
- [ ] Kafka is referenced in at least one application
- [ ] JSON Schema files exist at specified paths

## Step 4: Regenerate

```bash
make regenerate
```

## Step 5: Build

```bash
make generate    # Generates JSON Schema Go code if added
make build
```

Fix any errors and retry until build passes.

## Step 6: Summary

Tell the user:
- What was added
- Producer driver: `pkg/drivers/kafka/{name}/`
- Generated methods: `Publish{EventName}(ctx, message)` for each event
- OnlineConf paths:
  - `/{service}/kafka/{client}/brokers` — broker list
  - `/{service}/kafka/{client}/auth_type` — none, PLAIN, SCRAM-SHA-256, SCRAM-SHA-512
  - `/{service}/kafka/{client}/events/{event_name}/topic` — topic override
- Default topic name = event name (override per environment via OnlineConf)
