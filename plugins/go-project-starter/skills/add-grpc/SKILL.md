# Add gRPC Client to Existing Project

Interactive command to add a gRPC client (`buf_client`) to an existing go-project-starter project.

Note: go-project-starter currently only supports gRPC **clients**. Server-side gRPC (`buf_server`) is not yet implemented.

## Mandatory Steps

The task is NOT complete until ALL of these pass:
1. Read existing project.yaml
2. Interview the user
3. Modify project.yaml
4. Run `make regenerate` successfully
5. Run `make build` successfully (no compilation errors)

## Step 1: Read Existing Config

```bash
cat .project-config/project.yaml
```

Identify existing grpc transports, applications, and tools versions (especially `buf_version`).

## Step 2: Interview

Ask the user:

1. **Service name** — gRPC service name from .proto file (e.g., `UserService`)
2. **Short name** — short identifier for package naming (e.g., `user`). Optional.
3. **Proto file path** — where is the .proto file? Relative to config directory.
4. **Port** — gRPC port (default 9000). Must not conflict with existing.
5. **Instantiation** — `static` (connect at startup, default) or `dynamic` (connect per-request)?
6. **Local plugins?** — use local buf plugins (`buf_local_plugins: true`)? Recommended if buf is installed locally.
7. **Which application?** — add to existing app or new one?
8. **Optional?** — should the gRPC client be an optional dependency?

## Step 3: Modify Config

### Add to `grpc` section:

```yaml
grpc:
  # ... existing grpc transports ...
  - name: {service_name}
    short: {short_name}
    path: ./{proto_file}
    port: {port}
    generator_type: buf_client
    buf_local_plugins: true
```

For dynamic instantiation:
```yaml
grpc:
  - name: {service_name}
    short: {short_name}
    path: ./{proto_file}
    port: {port}
    generator_type: buf_client
    instantiation: dynamic
```

### Add to application:

```yaml
applications:
  - name: server
    transport:
      # ... existing transports ...
      - name: {service_name}
      # Or with optional flag:
      - name: {service_name}
        config:
          optional: true
```

### Validation checklist:
- [ ] gRPC name is unique
- [ ] Proto file exists at specified path
- [ ] Port doesn't conflict with existing transports
- [ ] `generator_type` is `buf_client`
- [ ] Transport added to at least one application
- [ ] `instantiation` is `static` or `dynamic` (if specified)

## Step 4: Regenerate

```bash
make regenerate
```

If fails, read error, fix config, retry.

## Step 5: Build

```bash
make generate    # Runs buf generate for proto files
make build
```

Common issues:
- Proto file not found — check path relative to configDir
- `buf` not installed — install buf or set `buf_local_plugins: false`
- Missing protoc plugins — install `protoc-gen-go` and `protoc-gen-go-grpc`

Fix any errors and retry until build passes.

## Step 6: Summary

Tell the user:
- Generated client location: `pkg/grpc/{short_name}/`
- buf config: `configs/transport/grpc/{service_name}/buf.gen.yaml`
- Proto spec copied to: `api/grpc/{service_name}/`

### Static mode:
- Client connects at startup
- Available via Service struct
- OnlineConf paths:
  - `/{service}/transport/grpc/{name}/host`
  - `/{service}/transport/grpc/{name}/port`

### Dynamic mode:
- Client NOT registered at startup
- NOT in Service struct
- Create via `NewDynamicClient(ctx, address)` — address passed at call time
- No OnlineConf needed — address is a parameter
- Use when target host is determined at runtime
