# Add Worker to Existing Project

Interactive command to add a daemon or queue worker to an existing go-project-starter project.

## Mandatory Steps

The task is NOT complete until ALL of these pass:
1. Read existing project.yaml
2. Interview the user
3. Modify project.yaml (and optionally create queues.yaml contract)
4. Run `make regenerate` successfully
5. Run `make build` successfully (no compilation errors)

## Step 1: Read Existing Config

```bash
cat .project-config/project.yaml
```

Identify:
- Existing applications, workers, transports
- Whether sys transport already exists
- Existing worker names (must be unique)

## Step 2: Interview

Ask the user:

1. **Worker type?** — `daemon` (periodic background tasks) or `queue` (message queue processor)
2. **Worker name** — unique name, e.g., `status_updater`, `email_processor`
3. **If queue:** queues contract — path to existing `queues.yaml` or create a new one? Since v0.24.0 `path:` accepts both local files and remote URIs (`git+ssh://`, `git+https://`, `https://`) — useful when contracts live in a shared repo. See main go-project-starter SKILL → "Remote Spec Sources" for the URI grammar. If creating a new contract:
   - Queue names (e.g., `email_send`, `report_generate`)
   - Description for each queue
   - Payload fields: name, type (`string`, `int`, `bool`, `float64`, `duration`), required flag
4. **Which application?** — add to existing app or create a separate container?

## Step 3: Modify Config

### Add worker section:

**Daemon:**
```yaml
worker:
  - name: {worker_name}
    generator_type: template
    generator_template: daemon
```

**Queue:**
```yaml
worker:
  - name: {worker_name}
    generator_type: template
    generator_template: queue
    path: [./queues.yaml]
```

### Create queues.yaml contract (if queue type and file doesn't exist):

```yaml
queues:
  - name: {queue_name}
    description: "{description}"
    payload:
      - name: {field_name}
        type: {type}
        required: true
```

### Add to existing application:
```yaml
applications:
  - name: server
    transport:
      - name: api
      - name: sys
    worker: [{worker_name}]          # Add worker
```

### Or create new application (separate container):
```yaml
applications:
  # ... existing apps unchanged ...
  - name: {app_name}
    transport:
      - name: sys      # sys MUST already exist in rest section
    worker: [{worker_name}]
```

### Validation checklist:
- [ ] Worker name is unique (no duplicates)
- [ ] Queue worker has `path` field pointing to existing contract file
- [ ] Worker is referenced in at least one application
- [ ] Application has at least one transport (sys)
- [ ] sys transport is defined in rest section

## Step 4: Regenerate

```bash
make regenerate
```

If fails, read error, fix config, retry.

## Step 5: Build

```bash
make build
```

If fails, fix and retry. Common issues:
- Missing sys transport definition
- Name conflicts
- Queue contract file not found

Only report success when build passes.

## Step 6: Summary

Tell the user:
- What was added to project.yaml
- Worker handler location: `internal/app/worker/{worker_name}/worker.go`
- Job logic location: `internal/app/worker/{worker_name}/job.go`

### Daemon-specific:

- OnlineConf paths:
  - `/{service}/worker/daemon/{name}/enabled` — true/false (DISABLED by default!)
  - `/{service}/worker/daemon/{name}/interval` — execution interval (e.g., "30s", "5m")
- Implementation pattern:
  ```go
  func (w *Worker) GetTasks(ctx context.Context, lastJobState any) ([]daemon.ITask, any, error) {
      enabled, _ := onlineconf.GetBool(ctx, GetConfigPath("enabled"), false)
      if !enabled {
          return nil, lastJobState, nil
      }
      interval, _ := onlineconf.GetDuration(ctx, GetConfigPath("interval"), 30*time.Second)
      // Check interval, return tasks
  }
  ```
- Workers are DISABLED by default — must set `enabled=true` in OnlineConf

### Queue-specific:

- OnlineConf paths:
  - `/{service}/worker/queue/{name}/batch_size` — items per batch
  - `/{service}/worker/queue/{name}/poll_interval` — queue poll interval
- Generated queue handler methods match contract queue names
- Run: `make run-{app_name}` (if separate container)
